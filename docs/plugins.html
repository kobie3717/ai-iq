Claude Code Plugin

Auto-capture errors and session snapshots with AI-IQ's Claude Code plugin. Persistent memory that works in the background.

What This Plugin Does

Once installed, AI-IQ will automatically:

  • ✓ Auto-capture errors — Failed Bash commands are automatically logged as error memories
  • ✓ Auto-snapshot sessions — When you stop working, AI-IQ detects git commits and file changes to create a session summary
  • ✓ Auto-export context — Regenerates MEMORY.md for Claude to read at the start of each session
  • ✓ Auto-maintain — Runs decay, garbage collection, and backups
  • ✓ Timeline tracking — Logs session start times for better temporal context

Quick Install

Install AI-IQ and run the automated installer:

# 1. Install AI-IQ
pip install ai-iq

# 2. Clone the repo (if not already cloned)
git clone https://github.com/kobie3717/ai-iq.git
cd ai-iq

# 3. Run the plugin installer
bash hooks/claude-code/install.sh

The installer will:

  • Check if memory-tool is installed
  • Copy hook scripts to ~/.claude/hooks/
  • Configure hooks in ~/.claude/settings.json
  • Optionally install a daily maintenance cron job

Hook Details

PostToolUse Hook (error-hook.sh)

  • Trigger: After every Bash tool invocation
  • Condition: Only activates on non-zero exit codes
  • Action: Captures command and error output, calls memory-tool log-error
  • Silent: Fails gracefully if memory-tool is not installed

Stop Hook (session-hook.sh)

  • Trigger: When Claude Code session ends
  • Actions:
    1. memory-tool auto-snapshot — Detects git/file changes and creates session summary
    2. memory-tool decay — Flags stale memories using FSRS-6 algorithm
    3. memory-tool export — Regenerates MEMORY.md with smart context
    4. memory-tool backup — Daily backup check (once per day)

SessionStart Hook (session-start-hook.sh)

  • Trigger: When Claude Code session starts
  • Action: Logs session start with 7-day expiry for timeline tracking
  • Silent: Fails gracefully if memory-tool is not installed

Manual Installation

If you prefer manual setup or need to merge with existing hooks:

1. Install AI-IQ

pip install ai-iq

2. Copy Hook Scripts

mkdir -p ~/.claude/hooks
cp hooks/claude-code/error-hook.sh ~/.claude/hooks/ai-iq-error-hook.sh
cp hooks/claude-code/session-hook.sh ~/.claude/hooks/ai-iq-session-hook.sh
cp hooks/claude-code/session-start-hook.sh ~/.claude/hooks/ai-iq-session-start-hook.sh
chmod +x ~/.claude/hooks/ai-iq-*.sh

3. Configure Hooks

Add to ~/.claude/settings.json (merge with existing hooks if needed):

{
  "hooks": {
    "Stop": [
      {
        "matcher": "",
        "hooks": [
          {
            "type": "command",
            "command": "bash ~/.claude/hooks/ai-iq-session-hook.sh"
          }
        ]
      }
    ],
    "PostToolUse": [
      {
        "matcher": "Bash",
        "hooks": [
          {
            "type": "command",
            "command": "bash ~/.claude/hooks/ai-iq-error-hook.sh"
          }
        ]
      }
    ],
    "SessionStart": [
      {
        "matcher": "",
        "hooks": [
          {
            "type": "command",
            "command": "bash ~/.claude/hooks/ai-iq-session-start-hook.sh"
          }
        ]
      }
    ]
  }
}

4. Add to Your Project's CLAUDE.md

Copy the AI-IQ section from CLAUDE.md.example to your project's CLAUDE.md:

cat hooks/claude-code/CLAUDE.md.example >> /your/project/CLAUDE.md

Daily Maintenance (Optional)

Install a daily cron job for background maintenance:

# Copy maintenance script
cp hooks/claude-code/daily-maintenance.sh ~/.claude/hooks/ai-iq-daily-maintenance.sh
chmod +x ~/.claude/hooks/ai-iq-daily-maintenance.sh

# Add cron job (runs at 3:17 AM daily)
(crontab -l 2>/dev/null; echo "17 3 * * * bash ~/.claude/hooks/ai-iq-daily-maintenance.sh") | crontab -

The daily maintenance runs:

  • memory-tool decay — Flag stale memories
  • memory-tool gc — Garbage collect old deleted memories (180 days)
  • memory-tool backup — Create SQLite backup
  • memory-tool export — Regenerate MEMORY.md

Logs are written to ~/.local/share/ai-memory/logs/daily-maintenance.log

Verifying Installation

Check if hooks are installed correctly:

# Check if hook scripts exist
ls -la ~/.claude/hooks/ai-iq-*.sh

# Check if memory-tool is installed
which memory-tool
memory-tool stats

# Test error capture (run in Claude Code)
bash -c "exit 1"  # Should create an error memory

# Test session snapshot (run in Claude Code, then stop session)
# Check with: memory-tool list --category session

How It Works in Practice

Scenario 1: Error Auto-Capture

# You run a command that fails
npm run build
# ERROR: Module not found: 'react-router-dom'

# AI-IQ automatically captures:
# Category: error
# Content: "Command failed: npm run build\nERROR: Module not found: 'react-router-dom'"
# Tags: npm, build
# Project: [current directory name]

# Later, you can search:
memory-tool search "react-router"
# → Returns the error with full context

Scenario 2: Session Snapshot

# During your session, you:
# - Fixed authentication bug
# - Added rate limiting
# - Committed: "feat: add rate limiting to auth endpoints"

# When you stop the session, AI-IQ automatically:
# 1. Detects git commit
# 2. Creates snapshot: "feat: add rate limiting to auth endpoints"
# 3. Runs decay to flag stale memories
# 4. Regenerates MEMORY.md with updated context

# Next session:
# Claude reads MEMORY.md and knows about your recent work

Scenario 3: Dream Consolidation

# You have multiple similar memories:
# #12: "Redis needs network_mode: host for Docker"
# #45: "Redis connection failed - needed network_mode: host"
# #67: "Docker Redis: use network_mode: host"

# Run dream mode (or it runs automatically in daily maintenance)
memory-tool dream

# Output:
# Consolidated 2 duplicates into #12
# #45 and #67 marked as superseded
# Kept newest memory with full context

Configuration

Customize AI-IQ behavior by editing hook scripts:

Error Hook Configuration

Edit ~/.claude/hooks/ai-iq-error-hook.sh:

  • Change error priority (default: 5)
  • Add custom tags
  • Filter which commands to capture

Session Hook Configuration

Edit ~/.claude/hooks/ai-iq-session-hook.sh:

  • Enable/disable auto-snapshot
  • Enable/disable decay
  • Enable/disable export
  • Configure backup frequency

Troubleshooting

Hooks Not Running

  • Check if scripts are executable: ls -la ~/.claude/hooks/ai-iq-*.sh
  • Verify ~/.claude/settings.json has correct hook configuration
  • Check Claude Code logs for hook errors

Memory-Tool Not Found

  • Verify installation: which memory-tool
  • Check Python path: python3 -m pip show ai-iq
  • Reinstall: pip install --force-reinstall ai-iq

Errors Not Being Captured

  • Verify PostToolUse hook is configured for "Bash" matcher
  • Test manually: bash ~/.claude/hooks/ai-iq-error-hook.sh
  • Check if exit code is being passed correctly
