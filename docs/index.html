Quick Start

Python API

from ai_iq import Memory

memory = Memory()

# Add memories
memory.add("User prefers dark mode", tags=["preference", "ui"])
memory.add("Redis bug fixed with network_mode: host", category="learning")

# Search (hybrid keyword + semantic)
results = memory.search("redis networking")
for r in results:
    print(f"#{r['id']}: {r['content']}")

# Update and delete
memory.update(1, "User STRONGLY prefers dark mode")
memory.delete(1)

Command Line

Python API Reference

Class: Memory

The main entry point for the AI-IQ memory system.

Memory(db_path: Optional[str] = None)

Initialize memory system with optional custom database path.

memory = Memory()  # Uses ./memories.db
memory = Memory("my_memories.db")  # Custom path

add(content, category="general", tags=None, project=None, priority=0, related_to=None, expires_at=None)

Add a new memory. Returns memory ID or None if duplicate.

Parameters:
  • content (str) — Memory content (required)
  • category (str) — One of: project, decision, preference, error, learning, pending, architecture, workflow, contact, general
  • tags (str | list) — Comma-separated tags or list
  • project (str) — Project name
  • priority (int) — Priority 0-10 (higher = more important)
  • related_to (int) — ID of related memory
  • expires_at (str) — Expiry date in YYYY-MM-DD format
mem_id = memory.add(
    "Use Redis for session storage",
    category="decision",
    tags=["redis", "caching"],
    project="MyApp",
    priority=8
)

search(query, mode="hybrid", include_token_estimate=False, compact=True)

Search memories with hybrid keyword + semantic search.

Parameters:
  • query (str) — Search query
  • mode (str) — "hybrid" (default), "keyword", or "semantic"
  • include_token_estimate (bool) — Add token estimates to results
  • compact (bool) — If True (default), truncate content to 100 chars
# Compact mode (default)
results = memory.search("redis caching")
for r in results:
    print(f"#{r['id']}: {r['content']}")  # Truncated

# Full mode
results = memory.search("redis", compact=False)
for r in results:
    print(r['content'])  # Full content

get(mem_id: int)

Get full details for a single memory by ID. Returns dict or None.

mem = memory.get(42)
if mem:
    print(mem["content"])

update(mem_id: int, content: str)

Update memory content. Auto-updates tags and re-generates embeddings.

memory.update(42, "Updated content here")

delete(mem_id: int)

Soft-delete a memory (marks as inactive, recoverable).

memory.delete(42)

list(category=None, project=None, tag=None, stale_only=False, expired_only=False)

List memories with optional filters. Returns list of dicts.

# Get all memories for a project
mems = memory.list(project="MyApp")

# Get all pending items
todos = memory.list(category="pending")

# Get stale memories that need review
stale = memory.list(stale_only=True)

stats()

Get database statistics. Returns dict with memory counts.

stats = memory.stats()
print(f"Total memories: {stats['total']}")
print(f"Active memories: {stats['active']}")
print(f"Categories: {stats['categories']}")

CLI Reference

Core Operations

Command Description
memory-tool add <category> "<content>" [options] Create memory with optional --project, --tags, --priority, --expires, --key
memory-tool update <id> "<content>" Update memory (auto re-embeds for semantic search)
memory-tool delete <id> Soft delete (recoverable)
memory-tool get <id> Full detail view with relationships
memory-tool list [filters] Filter with --project/--category/--tag/--stale/--expired

Search & Discovery

Command Description
memory-tool search "<query>" Hybrid search (keyword + semantic + graph)
memory-tool search "<query>" --full Verbose output with full content
memory-tool search "<query>" --semantic Semantic-only (vector similarity)
memory-tool search "<query>" --keyword Keyword-only (FTS5)
memory-tool conflicts Find 50-85% similar memories (potential duplicates)
memory-tool pending Show TODO items
memory-tool hot Most accessed memories (immune to decay)

Beliefs & Predictions

Command Description
memory-tool believe "<statement>" --confidence 0.7 Create belief with confidence score (0.01-0.99)
memory-tool predict "<prediction>" --based-on <id> --deadline YYYY-MM-DD Make testable prediction
memory-tool resolve <pred_id> --confirmed "<outcome>" Resolve prediction (auto-updates belief confidence)
memory-tool beliefs [--weak|--strong|--conflicts] List beliefs by confidence
memory-tool predictions [--open|--confirmed|--refuted] Filter predictions by status

Knowledge Graph

Command Description
memory-tool graph add <type> <name> [summary] Create entity (person/project/org/feature/concept/tool/service)
memory-tool graph rel <from> <type> <to> Add relationship (uses/depends_on/PREVENTS/RESOLVES/LEADS_TO)
memory-tool graph fact <entity> <key> <value> Set entity metadata (tracks history)
memory-tool graph get <name> Entity details with facts + relationships + linked memories
memory-tool graph spread <name> [depth] Spreading activation (find related context)
memory-tool graph auto-link Auto-link all memories to entities by keyword matching

Maintenance

Command Description
memory-tool dream Consolidate duplicates, resolve conflicts, normalize metadata
memory-tool decay Flag stale memories, reduce priorities, expire old TODOs
memory-tool next Smart suggestions: what needs attention now
memory-tool stats Full statistics (memories + vectors + graph + beliefs)
memory-tool backup Manual backup to ~/.claude/backups/
memory-tool restore <file> Restore from backup

Advanced Features

🎯 Belief System (Unique to AI-IQ)

Track what your AI thinks it knows with confidence scores (0.01-0.99). When predictions resolve, related beliefs strengthen or weaken automatically via Bayesian-style learning.

memory-tool believe "TypeScript reduces bugs vs JavaScript" --confidence 0.75
memory-tool predict "Next rewrite will have 30% fewer runtime errors" --based-on 1 --deadline 2026-06-01
memory-tool resolve 1 --confirmed "Runtime errors down 42%"  # Belief → 0.85
memory-tool beliefs --weak  # Show lowest confidence beliefs

💤 Dream Mode (Unique to AI-IQ)

Autonomous consolidation inspired by REM sleep in biological memory. Finds duplicates (85-95% similar), auto-merges with provenance, detects contradictions, and strengthens frequently-accessed patterns.

memory-tool dream
# Output:
#   Consolidated 5 duplicates (kept newer, marked old as superseded)
#   Detected 2 potential contradictions (flagged for review)
#   Normalized 8 dates to ISO format
#   Strengthened 12 high-access memories (+1 priority)

🔗 Knowledge Graph

Build relationships between entities and extract context via spreading activation. Supports causal edges (PREVENTS, RESOLVES, LEADS_TO, REQUIRES) for cause-effect reasoning.

memory-tool graph add project WhatsAuction "Real-time auction platform"
memory-tool graph add tool Redis "In-memory cache"
memory-tool graph rel WhatsAuction uses Redis "For session storage"
memory-tool graph rel WhatsAuction REQUIRES Redis "Cache must be up before app"
memory-tool graph spread WhatsAuction --depth 2  # Find all related context

🎭 Identity Layer (Unique to AI-IQ)

Discovers behavioral traits from memory patterns. Detects tendencies like prefers_docker, ships_fast, tests_first with evidence count and confidence tracking.

memory-tool identity --discover
memory-tool identity --snapshot "After WhatsAuction rewrite"
memory-tool identity --evolution prefers_docker  # Show trait changes over time

📖 Narrative Memory (Unique to AI-IQ)

Builds cause-effect stories from causal graph edges. Walks LEADS_TO, PREVENTS, RESOLVES, REQUIRES edges chronologically with linked context.

memory-tool narrative WhatsAuction

📊 Meta-Learning (Unique to AI-IQ)

Self-improving search that tracks which results you actually use. Auto-adjusts RRF fusion weights based on keyword vs semantic hit rates.

memory-tool search-quality  # View effectiveness metrics
memory-tool hot  # Show most accessed memories (immune to decay)
