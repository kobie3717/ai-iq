Full CLI and Python API documentation for AI-IQ persistent memory system.

Core Operations

Basic CRUD operations for creating, reading, updating, and deleting memories.

Command Description
memory-tool add <category> "<content>" [options] Create memory with optional --project, --tags, --priority, --expires, --key
memory-tool update <id> "<content>" Update memory (auto re-embeds for semantic search)
memory-tool delete <id> Soft delete (recoverable)
memory-tool get <id> Full detail view with relationships
memory-tool list [filters] Filter with --project/--category/--tag/--stale/--expired

Examples

# Add memories with metadata
memory-tool add decision "Chose PostgreSQL for JSONB support" --project BlogAPI --priority 8
memory-tool add error "CORS fix: add credentials: include" --tags cors,fetch

# Upsert-style with topic keys (updates instead of creating duplicates)
memory-tool add preference "Dark mode preferred" --key user_theme_preference
memory-tool add preference "Dark mode with high contrast" --key user_theme_preference  # Updates previous

# Set expiry for temporary TODOs
memory-tool add pending "Review PR #123" --expires 2026-04-15

# Update existing memory
memory-tool update 42 "Chose PostgreSQL for JSONB and better full-text search"

# Get full details
memory-tool get 42

# List with filters
memory-tool list --project BlogAPI
memory-tool list --category error
memory-tool list --tag cors
memory-tool list --stale           # Show stale memories
memory-tool list --expired         # Show expired memories

Search & Discovery

Hybrid search combining keyword (FTS5), semantic (vector), and graph traversal.

Command Description
memory-tool search "<query>" Hybrid search (default) — keyword + semantic + graph with RRF fusion
memory-tool search "<query>" --full Verbose output with full content and metadata
memory-tool search "<query>" --semantic Semantic-only (vector similarity using all-MiniLM-L6-v2)
memory-tool search "<query>" --keyword Keyword-only (FTS5 full-text search)
memory-tool conflicts Find 50-85% similar memories (potential duplicates)
memory-tool merge <id1> <id2> Merge similar memories (keeps newer)
memory-tool supersede <old> <new> Mark old as superseded by new
memory-tool pending Show TODO items (category=pending)
memory-tool hot Most accessed memories (access_count >= 5 = immune to decay)
memory-tool next Smart suggestions: what needs attention now

Examples

# Search with different modes
memory-tool search "redis cache"              # Hybrid (default)
memory-tool search "redis cache" --full       # Full details
memory-tool search "redis cache" --semantic   # Conceptual similarity
memory-tool search "redis cache" --keyword    # Exact word matching

# Find and resolve conflicts
memory-tool conflicts                         # Show potential duplicates
memory-tool merge 42 43                       # Merge #42 and #43 (keeps newer)
memory-tool supersede 42 43                   # Mark #42 as superseded by #43

# Review action items
memory-tool pending                           # Show all TODOs
memory-tool hot                               # Show frequently accessed memories
memory-tool next                              # Get smart suggestions

Beliefs & Predictions

Unique to AI-IQ: Track confidence-scored beliefs and testable predictions with Bayesian updates.

Command Description
memory-tool believe "<statement>" --confidence 0.7 Create belief with confidence score (0.01-0.99)
memory-tool predict "<prediction>" --based-on <id> --deadline YYYY-MM-DD Make testable prediction based on a belief
memory-tool resolve <pred_id> --confirmed "<outcome>" Resolve prediction as confirmed (strengthens related beliefs)
memory-tool resolve <pred_id> --refuted "<outcome>" Resolve prediction as refuted (weakens related beliefs)
memory-tool beliefs List all beliefs sorted by confidence
memory-tool beliefs --weak Show weakest beliefs (lowest confidence)
memory-tool beliefs --strong Show strongest beliefs (highest confidence)
memory-tool beliefs --conflicts Find contradicting beliefs
memory-tool predictions List all predictions
memory-tool predictions --open Open predictions only
memory-tool predictions --confirmed Confirmed predictions
memory-tool predictions --refuted Refuted predictions
memory-tool predictions --expired Past deadline but unresolved

Examples

# Create a belief
memory-tool believe "TypeScript reduces bugs vs JavaScript" --confidence 0.75 --project CompanyCodebase

# Make a testable prediction
memory-tool predict "Next rewrite will have 30% fewer runtime errors" --based-on 1 --deadline 2026-06-01 --expect "30% reduction"

# Resolve prediction (automatically updates belief confidence)
memory-tool resolve 1 --confirmed "Runtime errors down 42%"  # Belief #1 → 0.85 confidence
memory-tool resolve 2 --refuted "No measurable change"       # Belief #1 → 0.65 confidence

# Query beliefs and predictions
memory-tool beliefs                 # All beliefs
memory-tool beliefs --weak          # Beliefs < 0.5 confidence
memory-tool beliefs --strong        # Beliefs > 0.8 confidence
memory-tool beliefs --conflicts     # Contradicting beliefs

memory-tool predictions             # All predictions
memory-tool predictions --open      # Active predictions
memory-tool predictions --expired   # Past deadline

Knowledge Graph

Build relationships between entities with spreading activation retrieval and causal reasoning.

Command Description
memory-tool graph add <type> <name> [summary] Create entity (person/project/org/feature/concept/tool/service)
memory-tool graph rel <from> <type> <to> [note] Add relationship: knows, works_on, owns, depends_on, built_by, uses, related_to, PREVENTS, RESOLVES, LEADS_TO, REQUIRES
memory-tool graph fact <entity> <key> <value> Set entity metadata (auto-tracks history)
memory-tool graph get <name> Entity details with facts + relationships + linked memories
memory-tool graph list [type] List all entities or filter by type
memory-tool graph spread <name> [depth] Spreading activation (find related entities, default depth=2)
memory-tool graph link <mem_id> <entity> Link memory to entity manually
memory-tool graph auto-link Auto-link all memories to entities by keyword matching

Examples

# Create entities
memory-tool graph add project WhatsAuction "Real-time auction platform"
memory-tool graph add tool Redis "In-memory cache"
memory-tool graph add person Alice "Frontend lead"

# Add relationships (semantic edges)
memory-tool graph rel WhatsAuction uses Redis "For session storage"
memory-tool graph rel Alice works_on WhatsAuction "Lead developer"

# Add causal relationships (causal edges for reasoning)
memory-tool graph rel WhatsAuction REQUIRES Redis "Cache must be up before app starts"
memory-tool graph rel Redis PREVENTS SessionLoss "Keeps sessions in memory"

# Set facts
memory-tool graph fact Redis version "7.2.4"
memory-tool graph fact Redis port "6379"
memory-tool graph fact WhatsAuction status "production"

# Query graph
memory-tool graph get WhatsAuction           # Full details
memory-tool graph spread WhatsAuction 2      # Find related context (depth=2)
memory-tool graph list                       # All entities
memory-tool graph list tool                  # Tools only

# Link memories to entities
memory-tool graph link 42 WhatsAuction       # Link memory #42
memory-tool graph auto-link                  # Auto-link all memories

Identity & Narrative

Unique to AI-IQ: Discover behavioral traits and build cause-effect stories from memory patterns.

Command Description
memory-tool identity --discover Detect behavioral traits from memory patterns
memory-tool identity --snapshot "<desc>" Save current trait snapshot with description
memory-tool identity --evolution <trait> Show how a specific trait changed over time
memory-tool narrative <entity> Build cause-effect story from causal graph edges

Examples

# Identity layer
memory-tool identity --discover
# Output: Discovered traits: prefers_docker (8 instances), ships_fast (5), tests_first (3)

memory-tool identity --snapshot "After WhatsAuction rewrite"
# Saves current state for comparison

memory-tool identity --evolution prefers_docker
# Shows how 'prefers_docker' trait changed over time

# Narrative memory
memory-tool narrative WhatsAuction
# Builds chronological story: Redis REQUIRES startup → WhatsAuction uses Redis → deployment LEADS_TO scaling

Meta-Learning & Feedback

Unique to AI-IQ: Search improves from usage patterns, tracking which results you actually use.

Command Description
memory-tool search-quality View keyword/semantic effectiveness metrics
memory-tool feedback <search_id> <used_ids> Manual feedback logging (which results were useful)
memory-tool tune-weights Re-calculate RRF fusion weights from feedback data

Examples

# View search quality metrics
memory-tool search-quality
# Shows: keyword hit rate 73%, semantic hit rate 45%, RRF weights adjusted

# Provide manual feedback (which results you used)
memory-tool feedback 42 1,3,5   # Search #42 used memories #1, #3, #5

# Re-tune weights based on feedback
memory-tool tune-weights

Session Management

Track structured workflows with steps, outcomes, and timing.

Command Description
memory-tool snapshot "<summary>" [--project X] Manual session snapshot with summary
memory-tool auto-snapshot Auto-detect changes from git/filesystem
memory-tool run start "<task>" --agent <name> --project <name> Start tracked workflow run
memory-tool run step <id> "<description>" Add step to active run
memory-tool run complete <id> "<outcome>" Mark run as completed
memory-tool run fail <id> "<reason>" Mark run as failed
memory-tool run cancel <id> Cancel run
memory-tool run list [--status running|completed|failed] List runs with filters

Examples

# Manual snapshots
memory-tool snapshot "Fixed authentication bug, added rate limiting" --project BlogAPI
memory-tool auto-snapshot  # Detects git/file changes

# Tracked workflows
memory-tool run start "Fix CORS issue" --agent claude --project BlogAPI
# Returns run_id: 1

memory-tool run step 1 "Identified missing headers in API responses"
memory-tool run step 1 "Added Access-Control-Allow-Credentials header"
memory-tool run complete 1 "CORS fixed, all tests passing"

# Query runs
memory-tool run list
memory-tool run list --status completed
memory-tool run list --status failed

Maintenance

Automated and manual maintenance operations for memory health.

Command Description
memory-tool dream Consolidate duplicates, resolve conflicts, normalize metadata, run decay (REM-like)
memory-tool decay Flag stale memories, reduce priorities, expire old TODOs
memory-tool stale Review stale memories
memory-tool gc [days] Garbage collect inactive memories (default: 180 days)
memory-tool reindex Rebuild vector embeddings for all memories
memory-tool backup Manual backup to ~/.claude/backups/
memory-tool restore <file> Restore from backup
memory-tool stats Full statistics (memories + vectors + graph + beliefs + runs)

Examples

# Dream mode (autonomous consolidation)
memory-tool dream
# Output:
#   Consolidated 5 duplicates (kept newer, marked old as superseded)
#   Detected 2 potential contradictions (flagged for review)
#   Normalized 8 dates to ISO format
#   Expired 3 pending items past deadline
#   Strengthened 12 high-access memories (+1 priority)
#   Weakened 4 low-access memories (-1 priority)

# Manual maintenance
memory-tool decay        # Run decay logic manually
memory-tool stale        # Show stale memories for review
memory-tool gc 90        # Delete memories inactive for 90+ days
memory-tool reindex      # Rebuild vector embeddings

# Backup & restore
memory-tool backup
# Output: Backup saved to ~/.claude/backups/memory_20260402_153045.db

memory-tool restore ~/.claude/backups/memory_20260402_153045.db

# Statistics
memory-tool stats
# Shows: total memories, active, inactive, categories, vectors, graph entities, beliefs, predictions

Python API Reference

Complete Python API documentation for the Memory class.

Memory(db_path: Optional[str] = None)

Initialize memory system with optional custom database path.

from ai_iq import Memory

memory = Memory()                      # Uses ./memories.db
memory = Memory("my_memories.db")      # Custom path
memory = Memory("/path/to/db.sqlite")  # Absolute path

add(content, category="general", tags=None, project=None, priority=0, related_to=None, expires_at=None) → Optional[int]

Add a new memory. Returns memory ID or None if duplicate.

Parameters:

  • content (str) — Memory content (required)
  • category (str) — One of: project, decision, preference, error, learning, pending, architecture, workflow, contact, general
  • tags (str | list) — Comma-separated tags or list of tags
  • project (str) — Project name
  • priority (int) — Priority 0-10 (higher = more important)
  • related_to (int) — ID of related memory
  • expires_at (str) — Expiry date in YYYY-MM-DD format
mem_id = memory.add(
    "Use Redis for session storage",
    category="decision",
    tags=["redis", "caching"],
    project="MyApp",
    priority=8
)

search(query, mode="hybrid", include_token_estimate=False, compact=True) → List[Dict]

Search memories with hybrid keyword + semantic search.

Parameters:

  • query (str) — Search query
  • mode (str) — "hybrid" (default), "keyword", or "semantic"
  • include_token_estimate (bool) — Add token estimates to results
  • compact (bool) — If True (default), truncate content to 100 chars
# Compact mode (default) - minimal data transfer
results = memory.search("redis caching")
for r in results:
    print(f"#{r['id']}: {r['content']}")  # Content truncated

# Full mode - get all data
results = memory.search("redis", compact=False)
for r in results:
    print(r['content'])  # Full content

# Different search modes
results = memory.search("redis", mode="semantic")   # Semantic-only
results = memory.search("redis", mode="keyword")    # Keyword-only

get(mem_id: int) → Optional[Dict]

Get full details for a single memory by ID. Returns dict or None if not found.

mem = memory.get(42)
if mem:
    print(mem["content"])
    print(mem["tags"])
    print(mem["created_at"])

update(mem_id: int, content: str) → None

Update memory content. Auto-updates tags and re-generates embeddings.

memory.update(42, "Updated content here")

delete(mem_id: int) → None

Soft-delete a memory (marks as inactive, recoverable).

memory.delete(42)

list(category=None, project=None, tag=None, stale_only=False, expired_only=False) → List[Dict]

List memories with optional filters. Returns list of memory dicts.

# Get all memories for a project
mems = memory.list(project="MyApp")

# Get all pending items
todos = memory.list(category="pending")

# Get stale memories that need review
stale = memory.list(stale_only=True)

# Get memories with specific tag
tagged = memory.list(tag="redis")

stats() → Dict[str, Any]

Get database statistics including memory counts, categories, and vector embeddings.

stats = memory.stats()
print(f"Total memories: {stats['total']}")
print(f"Active memories: {stats['active']}")
print(f"Inactive memories: {stats['inactive']}")
print(f"Categories: {stats['categories']}")
print(f"Vector embeddings: {stats['vectors']}")
