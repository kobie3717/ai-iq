Quick Start Guide

Get up and running with AI-IQ in 5 minutes. Learn the basics of adding, searching, and managing AI memories.

1. Installation

Install AI-IQ from PyPI:

pip install ai-iq

Optional: For semantic search with vector embeddings, install the full version: 

pip install ai-iq[full]

Requirements: Python 3.8+ and SQLite 3.37+

2. Python API Basics

Create your first memory in 5 lines:

from ai_iq import Memory

memory = Memory()  # Creates memories.db in current directory

# Add a memory
mem_id = memory.add("User prefers dark mode", tags=["preference", "ui"])

# Search memories
results = memory.search("dark mode")
for r in results:
    print(f"#{r['id']}: {r['content']}")

# Output: #1: User prefers dark mode

3. CLI Usage Basics

The CLI is perfect for quick notes and automation:

# Add memories
memory-tool add preference "User likes Bitcoin" --tags crypto,finance
memory-tool add learning "Docker needs network_mode: host for Redis" --project MyApp

# Search memories (hybrid keyword + semantic)
memory-tool search "docker networking"

# View full details for a specific memory
memory-tool get 1

# List all memories for a project
memory-tool list --project MyApp

# Update a memory
memory-tool update 1 "User REALLY likes Bitcoin"

# Delete a memory (soft delete, recoverable)
memory-tool delete 1

4. Memory Categories

Organize memories with categories:

Category Use For Example
project Project-specific facts "WhatsAuction uses Redis for sessions"
decision Architecture choices "Chose PostgreSQL over MySQL for JSONB support"
preference User preferences "User prefers TypeScript over JavaScript"
error Bugs and solutions "Fixed CORS by adding credentials: include"
learning Lessons learned "Always validate JWT tokens server-side"
pending TODOs and action items "Add rate limiting to auth endpoints"
architecture System design "Microservices pattern with event bus"
workflow Process documentation "Deploy process: test → staging → prod"
contact People and organizations "Alice handles frontend deployments" 
# Examples
memory-tool add decision "Using Next.js for SSR performance" --project Blog
memory-tool add error "Redis connection failed - needed network_mode: host" --project DockerSetup
memory-tool add pending "Implement rate limiting" --expires 2026-04-30 --project API

6. Python Integration Example

Build a simple chatbot with memory:

from ai_iq import Memory

# Initialize memory
memory = Memory("chatbot_memory.db")

# Simulate conversation
user_input = "I prefer Python over JavaScript"

# Store preference
memory.add(
    user_input,
    category="preference",
    tags=["language", "coding"],
    priority=5
)

# Later, retrieve context
results = memory.search("programming language preference")
if results:
    context = results[0]['content']
    print(f"Remembered: {context}")
    # Use this context to personalize responses

# Update memory when user changes mind
memory.update(results[0]['id'], "I prefer Python for backend, JavaScript for frontend")

# Check stats
stats = memory.stats()
print(f"Total memories: {stats['total']}")
print(f"Active memories: {stats['active']}")
print(f"Categories: {stats['categories']}")

7. Search Modes

AI-IQ offers three search modes:

Mode How It Works Best For
hybrid (default) Combines keyword (FTS5) + semantic (vector) + graph traversal with RRF fusion Most queries — balances precision and recall
keyword Fast FTS5 full-text search, exact word matching Exact terms, technical keywords, code snippets
semantic Vector similarity using all-MiniLM-L6-v2 embeddings Conceptual queries, synonyms, paraphrased content 
# CLI
memory-tool search "redis cache"              # Hybrid (default)
memory-tool search "redis cache" --keyword    # Keyword-only
memory-tool search "redis cache" --semantic   # Semantic-only

# Python API
results = memory.search("redis cache")                    # Hybrid
results = memory.search("redis cache", mode="keyword")    # Keyword-only
results = memory.search("redis cache", mode="semantic")   # Semantic-only
