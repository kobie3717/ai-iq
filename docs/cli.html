CLI Reference

Complete command-line reference for the memory-tool CLI. Available after installing AI-IQ.

Core Operations

Command Description
memory-tool add <category> "<content>" Create memory with optional --project, --tags, --priority, --expires, --key
memory-tool update <id> "<new content>" Update memory (auto re-embeds for semantic search)
memory-tool delete <id> Soft delete (recoverable)
memory-tool get <id> Full detail view with relationships
memory-tool list Filter with --project/--category/--tag/--stale/--expired

Examples

# Add a learning memory
memory-tool add learning "Redis needs network_mode: host in Docker" --project MyApp --tags redis,docker

# Add a pending task with expiry
memory-tool add pending "Review API docs before launch" --expires 2026-06-01 --priority 8

# Update a memory
memory-tool update 42 "Redis ALWAYS needs network_mode: host"

# Get full details
memory-tool get 42

# List by project
memory-tool list --project MyApp

# List stale memories
memory-tool list --stale

Search & Discovery

Command Description
memory-tool search "<query>" Hybrid search (keyword + semantic + graph)
memory-tool search "<query>" --full Verbose output with all fields
memory-tool search "<query>" --semantic Semantic-only (vector similarity)
memory-tool search "<query>" --keyword Keyword-only (FTS5)
memory-tool conflicts Find 50-85% similar memories (potential duplicates)
memory-tool merge <id1> <id2> Merge similar memories (keeps newer)
memory-tool supersede <old> <new> Mark old as superseded by new
memory-tool pending Show TODO items
memory-tool hot Most accessed memories (access_count >= 5 = immune to decay)
memory-tool next Smart suggestions: what needs attention now

Examples

# Hybrid search (default)
memory-tool search "redis networking"

# Full verbose output
memory-tool search "redis" --full

# Find duplicates
memory-tool conflicts

# Merge duplicate memories
memory-tool merge 42 43

# See what needs attention
memory-tool next

Beliefs & Predictions

Command Description
memory-tool believe "<statement>" --confidence 0.7 Create belief with confidence score (0.01-0.99)
memory-tool predict "<prediction>" --based-on <id> Make testable prediction with --deadline YYYY-MM-DD
memory-tool resolve <pred_id> --confirmed "<outcome>" Resolve prediction (or --refuted), auto-updates belief confidence
memory-tool beliefs List beliefs with --weak/--strong/--conflicts filters
memory-tool predictions Filter with --open/--confirmed/--refuted/--expired

Examples

# Create a belief
memory-tool believe "TypeScript reduces bugs vs JavaScript" --confidence 0.75

# Make a prediction based on that belief
memory-tool predict "Next rewrite will have 30% fewer runtime errors" \\
    --based-on 1 --deadline 2026-06-01

# Resolve prediction (strengthens belief to ~0.85)
memory-tool resolve 1 --confirmed "Runtime errors down 42%"

# Show weak beliefs
memory-tool beliefs --weak

# Show open predictions
memory-tool predictions --open

Knowledge Graph

Command Description
memory-tool graph add <type> <name> Create entity (person/project/org/feature/concept/tool/service)
memory-tool graph rel <from> <type> <to> Add relationship (uses/depends_on/PREVENTS/RESOLVES/LEADS_TO/REQUIRES)
memory-tool graph fact <entity> <key> <value> Set entity metadata (tracks history)
memory-tool graph get <name> Entity details with facts + relationships + linked memories
memory-tool graph spread <name> Spreading activation (find related context, optional --depth)
memory-tool graph auto-link Auto-link all memories to entities by keyword matching
memory-tool graph list List entities with optional type filter

Examples

# Create entities
memory-tool graph add project WhatsAuction "Real-time auction platform"
memory-tool graph add tool Redis "In-memory cache"

# Create relationships
memory-tool graph rel WhatsAuction uses Redis "For session storage"
memory-tool graph rel WhatsAuction REQUIRES Redis "Cache must be up before app starts"

# Add facts
memory-tool graph fact Redis version "7.2.4"
memory-tool graph fact Redis port "6379"

# Spreading activation - find all related context
memory-tool graph spread WhatsAuction --depth 2

# Auto-link existing memories to entities
memory-tool graph auto-link

# Get entity details
memory-tool graph get Redis

Identity & Narrative

Command Description
memory-tool identity --discover Detect behavioral traits from memory patterns
memory-tool identity --snapshot "<desc>" Save current trait snapshot
memory-tool identity --evolution <trait> Show how trait changed over time
memory-tool narrative <entity> Build cause-effect story from causal graph edges

Examples

# Discover behavioral traits
memory-tool identity --discover

# Save snapshot
memory-tool identity --snapshot "After WhatsAuction rewrite"

# See trait evolution
memory-tool identity --evolution prefers_docker

# Build narrative
memory-tool narrative WhatsAuction

Maintenance

Command Description
memory-tool dream Consolidate duplicates, resolve conflicts, normalize metadata, run decay
memory-tool decay Flag stale memories, reduce priorities, expire old TODOs
memory-tool stale Review stale memories
memory-tool gc Garbage collect inactive memories (default: 180 days)
memory-tool reindex Rebuild vector embeddings for all memories
memory-tool backup Manual backup to ~/.claude/backups/
memory-tool restore <file> Restore from backup
memory-tool stats Full statistics (memories + vectors + graph + beliefs + runs)

Examples

# Run dream consolidation
memory-tool dream

# Check statistics
memory-tool stats

# Manual backup
memory-tool backup

# Garbage collection
memory-tool gc 90  # Delete inactive memories older than 90 days

Session Management

Command Description
memory-tool snapshot "<summary>" Manual session snapshot with optional --project
memory-tool auto-snapshot Auto-detect changes from git/filesystem
memory-tool run start "<task>" Start tracked workflow with --agent and --project
memory-tool run step <id> "<description>" Add step to active run
memory-tool run complete <id> "<outcome>" Finish run (or fail/cancel)
memory-tool run list List runs with --status running/completed/failed

Examples

# Manual snapshot
memory-tool snapshot "Completed WhatsAuction payment integration" --project WhatsAuction

# Track a workflow
memory-tool run start "Fix authentication bug" --agent claude --project MyApp
memory-tool run step 1 "Identified issue in JWT validation"
memory-tool run step 1 "Added test coverage"
memory-tool run complete 1 "Fixed, all tests passing"

# List completed runs
memory-tool run list --status completed

Categories

Valid categories for the add command:

  • project — Project-specific information
  • decision — Architecture or design decisions
  • preference — User or team preferences
  • error — Bugs, errors, and fixes
  • learning — Lessons learned, gotchas
  • pending — TODOs, action items
  • architecture — System architecture notes
  • workflow — Process and workflow notes
  • contact — People and contact info
  • general — Uncategorized (default)
